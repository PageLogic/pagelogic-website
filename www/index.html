    Selling points

    React, Vue and Angular strive to turn HTML authoring into JavaScript development. We do the opposite. PageLogic brings reactivity to HTML itself with a small set of extensions, making it ideal for presentation logic. Deliverying content-ready pages to search engines can be challenging. PageLogic does it by default. Code blocks can easily become reusable components. This promotes building up your own libraries.

    Why PageLogic

    • Zero boilerplate

      forget ceremonies like React's useState(), render() etc. PageLogic handles these details behind the scenes, keeping your code clean and expressive.

    • Zero dependencies

      compiled pages only need their accompanying JavaScript file and the PageLogic runtime, both generated by the compiler or the server.

    • Super lightweight

      it sports a tiny runtime of less than 4kB (minified, compressed).

    • No Virtual DOM

      PageLogic uses direct DOM reflection, a much lighter and efficient solution.

    • Isomorphic by design

      with traditional frameworks, running the same logic in both the server and the client can be difficult. PageLogic server does it by default.

    • Advanced routing

      the same set of pages can work as a classic website and as a modern SPA at the same time.

    • Simple components

      going from a simple block of code to a component is trivial. This makes it practical to build your own reusable libraries.

    • Flexible deployment

      PageLogic pages can be served using the included Node.js server or Express middleware, or can be compiled to static HTML.

    Roadmap

    PageLogic is still in its infancy and is currently in its first development release, v.1.x. Stable releases for production will have even numbers, starting with 2.x. First production release is on course for Spring 2024.
